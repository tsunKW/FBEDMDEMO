📝 可执行脚本:

   └─ 用法: upload.bat <token>

🚀 快速开始（3 步）
═══════════════════════════════════════════════════════════════════════════════

第 1 步: 准备 GitHub Token
────────────────────────────
访问: https://github.com/settings/tokens
  1. 点击 "Generate new token"
  2. 选择 "Generate new token (classic)"
  3. 勾选权限: ✓ repo 和 ✓ workflow
  4. 点击 "Generate token"
  5. 复制显示的 Token（只显示一次！）

第 2 步: 打开命令行
────────────────────────────
  1. 按 Win+R
  2. 输入: cmd
  3. 按 Enter

第 3 步: 执行上传
────────────────────────────
进入目录:
  cd /d e:\lab\台塑生醫旗艦館\FBEDM

执行命令（选择一个）:

选项 A - 使用批处理（推荐）:
  upload.bat your_github_token_here

选项 B - 直接使用 Python:
  python upload_files_with_token.py your_github_token_here

选项 C - 设置环境变量:
  set GITHUB_TOKEN=your_github_token_here
  python upload_to_github.py

例如:

🚨 常见问题和解决方案
═══════════════════════════════════════════════════════════════════════════════

Q1: "python 不是内部或外部命令"
───────────────────────────────
症状: 执行命令后出现此错误
原因: Python 未安装或未添加到 PATH
解决:
  1. 从 https://www.python.org/ 安装 Python
  2. 安装时勾选 "Add Python to PATH"
  3. 重启计算机
  4. 重新尝试

Q2: "requests 模块未找到"
───────────────────────────
症状: ModuleNotFoundError: No module named 'requests'
原因: requests 库未安装
解决:
  pip install requests

Q3: "401 Unauthorized"
───────────────────────────
症状: 上传时出现 401 错误
原因: Token 无效或已过期
解决:
  1. 检查 Token 是否完整（无空格）
  2. 检查 Token 前缀是否为 "ghp_"
  3. 访问 https://github.com/settings/tokens 检查 Token 状态
  4. 如果过期，删除并生成新 Token

Q4: "404 Not Found"
───────────────────────────
症状: 上传时出现 404 错误
原因: 仓库不存在或 Token 没有访问权限
解决:
  1. 确认仓库地址: https://github.com/tsunKW/FBEDMDEMO
  2. 确认 Token 有 "repo" 权限
  3. 确认对该仓库有写入权限

Q5: "422 Unprocessable Entity"
───────────────────────────
症状: 上传时出现 422 错误
原因: 文件内容冲突或编码问题
解决:
  - 脚本会自动重试（最多 3 次）
  - 如果仍然失败，检查文件编码（应为 UTF-8）

Q6: 上传速度很慢
───────────────────────────
症状: 每个文件上传耗时过长
原因: 网络连接慢或文件较大
解决:
  - 这是正常的，取决于网络速度
  - 预计 5-10 分钟上传 88 个文件
  - 检查网络连接

⚠️ 安全建议
═══════════════════════════════════════════════════════════════════════════════

🔴 绝对不要:
  ✗ 在脚本代码中硬编码 Token
  ✗ 将 Token 提交到版本控制（git）
  ✗ 在公开论坛或社交媒体分享 Token
  ✗ 在未加密的文件中保存 Token
  ✗ 在浏览器历史中留下 Token

🟡 谨慎:
  ⚠️ Token 只显示一次，请妥善保管
  ⚠️ 定期审查 Token 权限
  ⚠️ 不使用时删除旧 Token
  ⚠️ 使用环境变量而不是命令行参数存储敏感信息（生产环境）

🟢 推荐做法:
  ✓ 使用命令行参数传递 Token（本地开发）
  ✓ 使用环境变量传递 Token（自动脚本）
  ✓ 使用 .gitignore 防止 Token 泄露
  ✓ 定期更换 Token
  ✓ 如果 Token 泄露立即删除
