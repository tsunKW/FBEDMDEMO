📝 前置要求
═════════════════════════════════════════════════════════════════════════════

✓ Python 3.6 或更高版本
  检查: python --version
  下载: https://www.python.org/

✓ 安装 requests 库
  命令: pip install requests

✓ 有效的 GitHub Personal Access Token
  获取方式:
    1. 访问 https://github.com/settings/tokens
    2. 点击 "Generate new token"
    3. 选择 "Generate new token (classic)"
    4. 勾选权限: repo 和 workflow
    5. 生成并复制 Token

⚠️ 安全提示
═════════════════════════════════════════════════════════════════════════════

🔴 重要:
  • 不要在脚本中硬编码 Token
  • 不要将 Token 提交到版本控制
  • 不要在公开场所分享 Token
  • Token 只显示一次，妥善保管
  • 定期审查 Token 权限
  • 如果 Token 泄露，立即删除并重新生成

🆘 故障排查
═════════════════════════════════════════════════════════════════════════════

问题: "python 不是内部或外部命令"
解决: 1. 安装 Python
     2. 检查 Python 是否在 PATH 中
     3. 使用完整路径: C:\Python39\python.exe upload_files_with_token.py token

问题: "requests 模块未找到"
解决: pip install requests

问题: "401 Unauthorized"
解决: 1. 检查 Token 是否正确
     2. 检查 Token 是否过期
     3. 确认 Token 有 repo 权限

问题: "仓库不存在"
解决: 确认仓库是 https://github.com/tsunKW/FBEDMDEMO

问题: "权限被拒绝"
解决: 1. 检查 Token 权限
     2. 检查是否有写入权限
     3. 如果是分支保护，临时禁用
